# fix(cmdstate): don't store execSetup and environment in state (#478)

## Summary

Stop persisting exec setup and environment to state to reduce state file bloat

## Commit Message

Per #411, the environment (and other task data) can get quite large for
exec objects. Given that exec commands are one-shot and only relevant
for the current run of Pebble, there's no need to persist them.

Note that none of this data is in "api-data" so it's not accessible via
the API at all right now, and so this is a non-breaking change.

Later if we want to make exec tasks a bit more introspectable we can add
the command line or other fields to "api-data" (but we shouldn't add the
entire environment).

Fixes #411.
