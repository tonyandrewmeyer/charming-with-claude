# fix: support "app/leader" unit syntax for run and exec (#216)

**Repository**: jubilant
**Commit**: [b170a09e](https://github.com/canonical/jubilant/commit/b170a09ee4e26fdfb69ddaa1562952e046c98ad4)
**Date**: 2025-10-30T14:16:10+01:00

## Classification

| Field | Value |
|-------|-------|
| Bug Area | juju-cli-wrapper |
| Bug Type | edge-case |
| Severity | high |
| Fix Category | source-fix |

## Summary

Using unit='app/leader' for run() or exec() crashed because Juju returns key as 'app/0' not 'app/leader'

## Commit Message

Currently if you specify unit='app/leader' to run() or exec() the
operation works, but the returned dict comes through with key 'app/0'
or similar, and Jubilant fails when looking for the key 'app/leader'.
Update the code to ignore the returned key.

Fixes #214
