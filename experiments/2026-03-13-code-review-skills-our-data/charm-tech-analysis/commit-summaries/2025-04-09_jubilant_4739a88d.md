## Summary

Breaking API change: remove_unit changed from string-or-iterable to *args for consistency with exec()

## Commit Message

This PR changes the signature of `Juju.remove_unit` to use `*args`
instead of distinguishing between a string or an iterable. This matches
the approach for `Juju.exec`.

I've added an extra unit test to guard against an off-by-one error.
