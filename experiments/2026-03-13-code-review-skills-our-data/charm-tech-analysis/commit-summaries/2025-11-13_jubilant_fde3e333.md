## Summary

Juju.exec machine parameter only accepted int, but LXD container IDs like '0/lxd/0' require str

## Commit Message

Changes the type annotation for the `machine` parameter in `Juju.exec`
method from `int` to `int | str` to support both integer machine IDs
(for convenience) and string machine IDs (for LXD containers).
