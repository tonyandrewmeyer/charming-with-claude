## Summary

don't cache secrets in ops

## Commit Message

This PR removes the "shared secret metadata cache".

Specific changes:
- direct hook command calls on Juju 3.6 and later
- read-modify-write sequence for Juju 3.5 and earlier
- the logic moved from model to the model backend
- unit tests updated
- integration tests pass as is

Open questions and comments:
- what other areas should be tested?
- what kinds of additional integration tests could be helpful?
- please check Secret object instantiation carefully

Ref #1536
