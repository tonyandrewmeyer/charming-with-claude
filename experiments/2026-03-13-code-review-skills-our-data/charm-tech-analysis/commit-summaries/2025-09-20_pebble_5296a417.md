## Summary

Fix flaky TestFailures check tests by increasing threshold and using more reliable timing

## Commit Message

TestFailuresBelowThreshold was failing with this error:

```
FAIL: manager_test.go:305: ManagerSuite.TestFailuresBelowThreshold

manager_test.go:338:
    c.Assert(lastTaskLog(s.overlord.State(), check.ChangeID), Matches, ".* INFO succeeded after 1 failure")
... value string = "2025-09-18T18:51:49Z INFO succeeded after 2 failures" ... regex string = ".* INFO succeeded after 1 failure"

2025-09-18T18:51:49.111Z [test] Check "chk1" failure 1/3: exit status 1 2025-09-18T18:51:49.131Z [test] Check "chk1" failure 2/3: exit status 1 2025-09-18T18:51:49.150Z [test] Check "chk1" failure 1/3: exit status 1 2025-09-18T18:51:49.243Z [test] Check "chk1" failure 1/1: check timed out after 25ms 2025-09-18T18:51:49.243Z [test] Check "chk1" threshold 1 hit, triggering action and recovering 2025-09-18T18:51:49.243Z [test] Change 1 task (Perform exec check "chk1") failed: check timed out after 25ms
```

This change makes that test more reliable on a loaded system (not
looking for exact number of failures), and also updates TestFailures
similarly.
