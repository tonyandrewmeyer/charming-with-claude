## Summary

replace git reference injection with wheel artifacts in charm test workflows

## Commit Message

For our compatibility tests, instead of patching the dependencies to
require the branch's version of ops[...] via a Git link, build local
wheels from the branch and patch the requirements to use that. This
avoids races where the appropriate revision of the branch is not yet
available from the CI runner.

Changes:
* [x] Create a reusable workflow to build ops wheels (ops, ops-scenario,
ops-tracing)
* [x] Update observability-charm-tests.yaml to use wheel artifacts
instead of git references
* [x] Update published-charms-tests.yaml to use wheel artifacts instead
of git references
* [x] Update db-charm-tests.yaml to use wheel artifacts instead of git
references
* [x] Update hello-charm-tests.yaml to use wheel artifacts instead of
git references
