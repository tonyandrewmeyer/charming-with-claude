## Commit Message

I clearly hadn't tested the entire command when I introduced it in
https://github.com/canonical/pebble/commit/e22f8beee34354779db51b5aba332aac40bc54db

The issue would show up in GitHub Actions logs as follows (but was
reproducible locally by copying the bash commands too):

  revision="$(snapcraft status pebble \
    --track latest --arch amd64 \
    | awk 'NR==1 { pos = index($0, "Revision") }  /edge/ { print 0+substr($0, pos) }'"

  echo "revision=$revision" >> $GITHUB_OUTPUT
  shell: /usr/bin/bash -e {0}
  env:
    SNAP_NAME: pebble
    DEFAULT_TRACK: latest
    DEFAULT_RISK: edge
    TO_TRACK: latest
    TO_RISK: candidate
    SNAPCRAFT_STORE_CREDENTIALS: ***
/home/runner/work/_temp/63db1d7a-1251-4a37-ba4a-0bfee8709f04.sh: line 5: unexpected EOF while looking for matching `"'
