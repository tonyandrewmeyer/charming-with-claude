## Summary

Show error details in check logs when failure threshold is reached by tracking previous change ID

## Commit Message

This PR enhances health check observability by tracking the previous
change associated with a check, propagating it through the
daemon/client/CLI layers, and improving logging when a check’s failure
threshold is reached.

Changes:

* Extend internal check tracking to include `PrevChangeID`, wire it
through `CheckManager`, daemon API (`/v1/checks` and refresh), Go
client, and both CLI commands (`pebble checks` and `pebble check`), and
document it in the OpenAPI spec.
* Improve task logging when a check hits its failure threshold by
returning an error enriched with `errorDetails(err)`, ensuring detailed
output is visible in task logs.
* Update tests and helper functions to normalise / ignore dynamic change
IDs (now including `prev-change-id`) where appropriate.

On the API side, the PR changes checks so that the previous change (if
there is one) is included in the check data (and this included in the
check/s response). On the CLI side, if there is no log data, but there
is a previous change, then pull the log data from the previous change.

Previously, `pebble checks` looked like this when failures==threshold:

```
Check  Level  Startup  Status  Successes  Failures  Change
chk1   -      enabled  down    ?          3/3       2
```

And `pebble check chk1` looked like this:

```
name: chk1
startup: enabled
status: down
failures: 3
threshold: 3
change-id: "2"
```

Now `pebble checks` looks like this when failures==threshold:

```
Check  Level  Startup  Status  Successes  Failures  Change
chk1   -      enabled  down    0          3/3       44 (exit status 2; ls: cannot access '/t... run "pebble tasks 43" for more)
```

And `pebble check chk1` looks like this:

```
name: chk1
startup: enabled
status: down
successes: 0
failures: 3
threshold: 3
change-id: "44"
prev-change-id: "43"
logs: |
    2026-02-03T18:32:16+13:00 ERROR exit status 2; ls: cannot access '/tmp/veryveryveryveryveryveryverylonglonglonglonglonglongname': No such file or directory
    2026-02-03T18:32:18+13:00 ERROR exit status 2; ls: cannot access '/tmp/veryveryveryveryveryveryverylonglonglonglonglonglongname': No such file or directory
    2026-02-03T18:32:20+13:00 ERROR exit status 2; ls: cannot access '/tmp/veryveryveryveryveryveryverylonglonglonglonglonglongname': No such file or directory
```
