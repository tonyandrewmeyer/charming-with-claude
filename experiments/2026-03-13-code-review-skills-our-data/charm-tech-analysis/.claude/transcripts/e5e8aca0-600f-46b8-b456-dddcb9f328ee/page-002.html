   614→### 6.4 False-Positive Controls Needed
   615→
   616→1. **Mutability pattern**: Not every dict assignment needs `.copy()`. Only flag when:
   617→   - The dict is received as a function parameter AND stored as an instance attribute
   618→   - The dict is an internal attribute AND returned from a public method
   619→   - The class is part of a testing framework or API boundary
   620→
   621→2. **None guard pattern**: Not every attribute access needs a None check. Only flag when:
   622→   - The value comes from an external source (Juju API, environment variable, dict lookup)
   623→   - The code is in an event handler (where Juju state may be incomplete)
   624→   - The value is documented as Optional in type annotations
   625→
   626→3. **Dict `.get()` pattern**: Not every `dict[key]` is a bug. Only flag when:
   627→   - The dict comes from parsing external data (JSON, YAML, CLI output)
   628→   - The key is a string literal that could vary by version or context
   629→   - There's no preceding `if key in dict` check
   630→
   631→4. **Go nil map pattern**: Only flag when:
   632→   - The map is a struct field (not a local variable)
   633→   - There's no initialization in the constructor or preceding code path
   634→   - The struct could be created via zero-value (`&Type{}`)
   635→
   636→5. **Lock ordering pattern**: Requires call-graph analysis to detect. Should only flag when:
   637→   - Two or more locks are acquired in the same function
   638→   - The same locks appear in different order in different functions
   639→   - The locks are struct-level mutexes (not function-scoped)
