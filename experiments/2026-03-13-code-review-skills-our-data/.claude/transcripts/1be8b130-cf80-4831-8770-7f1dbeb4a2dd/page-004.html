Noted — Go repos (pebble, concierge) last. Let me give my assessment of the operator supplementary findings first.

Operator — my assessment

H-1: Service.to_dict() / _merge() drop user-id: 0 — Real bug. Confirmed at line 1011: {name: value for name, value in fields if value} drops any falsy value including 0. UID/GID 0 is root, a valid and important value. The _merge at line 1021 has the same issue with if not value. This silently corrupts Pebble layer configuration. Strong PR candidate.

H-2: _event_context() lacks try/finally — Real bug. Confirmed at lines 948-956: the yield has no try/finally, so if a handler raises, _hook_is_running and _event_name are never restored. In production the process typically exits on exception so the stale state is harmless. But in Harness (line 437, 1398) and in _reemit (line 1019 loops over multiple observers), an exception in one handler leaves stale state for anything that runs afterward. The codebase already has the correct pattern at _prevent_recursion() (line 3555). Strong PR candidate.

M-1: Scenario secret_get() returns without copy — Real bug. Confirmed at lines 484/486. Production Secret.get_content() returns self._content.copy(). Scenario returns direct references to latest_content and tracked_content. Same class as the relation_get copy bug we already fixed (PR #2376). PR candidate.

M-2: Scenario action_get() returns without copy — Real bug. Confirmed at line 653: returns action.params directly. Production returns fresh data from Juju each time; Harness builds a new dict each time. Scenario returns the shared reference. PR candidate — could bundle with M-1.

M-3: Scenario _render_services() mutates Layer objects — Real bug. Confirmed at lines 1087-1095. Line 1094 stores a direct reference to a layer's Service, then line 1092 calls _merge() on it, mutating the original Layer's Service in place. Calling .plan twice accumulates duplicates in list fields (after, before, requires). Same pattern applies to _render_checks() and _render_log_targets(). This is a variant of the bug fixed in commit 3dda5b5f. PR candidate.

L-1: Plan.services/checks/log_targets return internal dicts — Minor. The docstring says "read-only" but the dict is mutable. In practice get_plan() creates new Plan objects each time, limiting impact. Skip.

L-2: Check._merge_exec/tcp falsy confusion — Minor. The values for exec/tcp config fields are strings and dicts — 0 or False are not realistic values. Skip unless bundled with H-1.

Summary for operator: - PR-worthy: H-1, H-2, M-1+M-2 (bundle), M-3 - Skip: L-1, L-2

That's potentially 4 PRs (or 3 if you bundle M-1+M-2). Do you agree with this assessment, or want to discuss any of these before I move on to charmlibs?
