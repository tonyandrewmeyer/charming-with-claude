#102026-03-15T06:00:20.590Z

let's talk anout #2377. is this definitely required even though we are using tempfile.temporarydirectory? explain to me why.

Good question. tempfile.TemporaryDirectory() creates the directory itself with restrictive permissions (0o700), so only the owner can list or traverse into it. That means the files inside are already inaccessible to other users regardless of individual file permissions — they can't even stat the path to reach the files.

So the answer is: no, this isn't a real vulnerability. The temp directory's 0o700 permissions already protect the files inside. The audit finding was based on looking at the open() call in isolation without considering that the parent directory is access-controlled.

The only scenario where it would matter is if someone changed the code to use a shared temp directory instead of tempfile.TemporaryDirectory(), but that's speculative.

Should I close #2377?
