## Commit Message

```
fix: ensure `ops.Pebble.pull` cleans up temporary files if it errors (#2087)

This PR moves the `_FilesParser.remove_files()` call in
`ops.pebble.Client.pull` into the `finally` block of a `try ...
finally`. [View the diff with whitespace changes
hidden](https://github.com/canonical/operator/pull/2087/files?diff=split&w=1)
to see what's actually changing here. This ensures that temporary files
are cleaned up even if exceptions are raised from the `try` block.

Fixes https://github.com/canonical/operator/issues/1440
```
