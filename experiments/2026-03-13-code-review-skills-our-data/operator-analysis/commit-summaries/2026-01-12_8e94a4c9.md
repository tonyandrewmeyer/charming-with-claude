## Commit Message

```
fix: use parse_rfc3339 for datetime parsing to support Python 3.10 (#2264)

Python 3.10's `datetime.fromisoformat()` requires fractional seconds to
be exactly 3 or 6 digits. Juju 4.0 returns timestamps with arbitrary
precision (e.g., `2026-01-05T23:34:25.50029526Z` with 8 digits), causing
parse failures.

## Changes

- **`ops/hookcmds/_utils.py`**: Modified `datetime_from_iso()` to use
existing `parse_rfc3339` from `ops._private.timeconv`, which handles
arbitrary precision fractional seconds. Also added support for space as
the date and time separator, which is required for `goal-state` (and
possibly other Juju fields, since it's a common format method).

- **`test/test_hookcmds.py`**: Added parametrized test covering edge
cases: no fractional seconds, 8 digits, 5 digits, 1 digit, 9 digits
(nanosecond precision).

- Renamed `datetime_to_iso` to `datetime_to_rfc3339` to be clearer what
it's doing.

## Example

```python
# Before: Fails on Python 3.10
datetime_from_iso('2026-01-05T23:34:25.50029526Z')  # ValueError

# After: Uses parse_rfc3339 which handles arbitrary precision
datetime_from_iso('2026-01-05T23:34:25.50029526Z')  # 2026-01-05 23:34:25.500295+00:00
datetime_from_iso('2026-04-10T18:34:45.65844+00:00')  # 2026-04-10 18:34:45.658440+00:00
```

This solution reuses existing, well-tested code from the repository
rather than reimplementing datetime parsing logic.
