## Commit Message

```
fix: prevent `KeyError` on `auth-type` when creating `CloudCredential` object (#2268)

Fixes #2267

When the hook command `credential-get` is executed as part of ops'
`model.get_cloud_spec()`, it currently fails with `KeyError` (see issue
linked above).

This PR drops the use of `dataclasses.asdict` to get a Juju-style
dictionary (to then convert into ops.model classes), since that doesn't
undo the `-` to `_` that happened when making the data classes. Instead,
new private methods to convert from a hookcmds class are added and used.

---------

Co-authored-by: Tony Meyer <tony.meyer@gmail.com>
```
