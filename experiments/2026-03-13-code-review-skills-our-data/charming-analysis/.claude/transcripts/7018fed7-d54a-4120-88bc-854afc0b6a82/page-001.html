# CLI-as-First-Class API Client Pattern

## Architecture Pattern

The author describes a unified binary approach where the server and client are distributed as a single executable. This eliminates separate client/server binaries and consolidates the API surface.

## CLI and Backend Relationship

The Brewlog binary functions as both server (hosting the API and web UI) and API client. The CLI commands provide programmatic access to the same endpoints exposed via HTTP, creating consistency across interfaces.

## Key Design Principles

**Single Surface Area**: "If you're building an API-backed app, I highly recommend this pattern. It keeps the surface area small" — maintaining one implementation rather than duplicating logic.

**Concurrent Development**: The CLI pattern enables scriptable workflows. The author references a bootstrap script that exercises the API, demonstrating how developers can automate setup and troubleshooting.

**API-First Enforcement**: The design requires that whenever HTTP endpoints are added, corresponding CLI commands ship simultaneously. This prevents API-only features and ensures complete coverage.

**Practical Benefits**: The unified binary simplifies deployment, backup, and introspection. Users can interact with the application through shell scripts, automation tools, or direct HTTP clients interchangeably.

This pattern emphasizes **consistency**, **discoverability**, and **scriptability** — treating the CLI as a first-class citizen rather than an afterthought.
