#### [BUG-001] Pebble -- Non-string values (bytes) in Pebble environment dict for TLS env vars (High)
- **Location**: `src/charm.py:236-237`
- **Pattern**: AP-005
- **Issue**: `b64encode()` returns `bytes`, not `str`. The Pebble environment dict values `SERVE_API_TLS_CERT_BASE64` and `SERVE_API_TLS_KEY_BASE64` are `bytes` objects (e.g., `b'dGVzdA=='`), not strings. Pebble expects all environment values to be strings and will reject or mishandle `bytes` objects.
- **Impact**: When TLS certificates are available, the Pebble layer will contain `bytes` values in the environment dict. This causes Pebble to fail to apply the layer or to produce incorrect environment variable values, breaking TLS configuration for the Oathkeeper workload.
- **Evidence**:
  ```python
  extra_env.update({
      "SERVE_API_TLS_CERT_BASE64": b64encode(bytes(self.cert_handler.cert, "utf-8")),
      "SERVE_API_TLS_KEY_BASE64": b64encode(bytes(self.cert_handler.key, "utf-8")),
  })
  ```
- **Recommended fix**:
  ```python
  extra_env.update({
      "SERVE_API_TLS_CERT_BASE64": b64encode(bytes(self.cert_handler.cert, "utf-8")).decode("ascii"),
      "SERVE_API_TLS_KEY_BASE64": b64encode(bytes(self.cert_handler.key, "utf-8")).decode("ascii"),
  })
  ```
- **Historical precedent**: Similar `bytes` vs `str` issues found in hydra-operator and SD-Core charms.

#### [BUG-002] Pebble -- Non-string values (bool, int) in Pebble environment dict for tracing env vars (High)
- **Location**: `src/charm.py:243-249`
- **Pattern**: AP-005
- **Issue**: When tracing is enabled, several environment values are Python `True` (bool) and `1` (int) instead of strings. `True` becomes `"True"` (capital T) when stringified, but Go/Oathkeeper expects lowercase `"true"`. The integer `1` is also not a string.
- **Impact**: Pebble may reject the layer, or the environment variables will contain incorrect values (e.g., `"True"` instead of `"true"` for `TRACING_ENABLED`), causing Oathkeeper to fail to initialise tracing or to misparse the configuration.
- **Evidence**:
  ```python
  extra_env.update({
      "TRACING_ENABLED": True,
      "TRACING_PROVIDER": "otel",
      "TRACING_PROVIDERS_OTLP_SERVER_URL": self._get_tracing_endpoint_info(),
      "TRACING_PROVIDERS_OTLP_INSECURE": True,
      "TRACING_PROVIDERS_OTLP_SAMPLING_SAMPLING_RATIO": 1,
  })
  ```
- **Recommended fix**:
  ```python
  extra_env.update({
      "TRACING_ENABLED": "true",
      "TRACING_PROVIDER": "otel",
      "TRACING_PROVIDERS_OTLP_SERVER_URL": self._get_tracing_endpoint_info(),
      "TRACING_PROVIDERS_OTLP_INSECURE": "true",
      "TRACING_PROVIDERS_OTLP_SAMPLING_SAMPLING_RATIO": "1",
  })
  ```
- **Historical precedent**: Identical pattern found in hydra-operator (bool in env dict). This is one of the most common Pebble-related bugs across charm repositories.

#### [BUG-003] Status -- BlockedStatus from _restart_service overwritten by ActiveStatus in caller (Medium)
- **Location**: `src/charm.py:575-577`
- **Pattern**: AP-050
- **Issue**: `_handle_status_update_config` calls `_restart_service()` which may set `BlockedStatus("Failed to restart the container, please consult the logs")` on failure and return. However, the caller unconditionally sets `ActiveStatus()` on line 577, overwriting the BlockedStatus. The operator never sees the restart failure.
- **Impact**: When the container fails to restart (e.g., due to Pebble rate limiting), the charm reports `ActiveStatus` instead of `BlockedStatus`, hiding the error from operators. The workload may be down while `juju status` shows everything is fine.
- **Evidence**:
  ```python
  def _restart_service(self) -> None:
      self._container.add_layer(self._container_name, self._oathkeeper_layer, combine=True)
      try:
          self._container.restart(self._container_name)
      except ChangeError as err:
          logger.error(str(err))
          self.unit.status = BlockedStatus(
              "Failed to restart the container, please consult the logs"
          )
          return

  def _handle_status_update_config(self, event: HookEvent) -> None:
      # ...
      self._restart_service()
      self.unit.status = ActiveStatus()  # Overwrites BlockedStatus!
  ```
- **Recommended fix**:
  ```python
  def _restart_service(self) -> bool:
      """Build a new layer and restart the service. Returns True on success."""
      self._container.add_layer(self._container_name, self._oathkeeper_layer, combine=True)
      try:
          self._container.restart(self._container_name)
      except ChangeError as err:
          logger.error(str(err))
          self.unit.status = BlockedStatus(
              "Failed to restart the container, please consult the logs"
          )
          return False
      return True

  def _handle_status_update_config(self, event: HookEvent) -> None:
      # ...
      self._update_config()
      if not self._restart_service():
          return
      self.unit.status = ActiveStatus()
  ```
- **Historical precedent**: Identical pattern found in temporal-k8s-operator (`_handle_frontend_tls` sets BlockedStatus but `_update` continues).

#### [BUG-004] Security -- Private key written with default (world-readable) permissions (Medium)
- **Location**: `src/charm.py:536-537`
- **Pattern**: AP-019
- **Issue**: The TLS private key is written to `/usr/local/share/ca-certificates/server.key` using `open(path, "w+")` without restricting file permissions. The default `umask` typically results in mode `0644` (world-readable). The private key should not be readable by other users or processes on the charm unit.
- **Impact**: The TLS private key is world-readable on the charm unit's filesystem. Any process running on the same unit (or any compromised workload with filesystem access) can read the private key, enabling TLS impersonation.
- **Evidence**:
  ```python
  with open(SERVER_KEY_PATH, "w+") as f:
      f.write(key)
  ```
- **Recommended fix**:
  ```python
  import stat

  # Write key with restricted permissions
  fd = os.open(SERVER_KEY_PATH, os.O_WRONLY | os.O_CREAT | os.O_TRUNC, 0o600)
  with os.fdopen(fd, "w") as f:
      f.write(key)
  ```
- **Historical precedent**: Common pattern across charm repositories; documented in the auth/identity bug patterns reference.
