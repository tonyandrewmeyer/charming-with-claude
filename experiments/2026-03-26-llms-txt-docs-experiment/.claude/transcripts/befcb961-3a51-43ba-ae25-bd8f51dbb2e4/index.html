Two excellent findings. Let me summarise:

1. Accept: text/markdown (Content Negotiation)

This would work out of the box — Claude Code already sends Accept: text/markdown as its preferred format. We don't need to tell the agent anything.

The key discovery: Claude Code's WebFetch sends Accept: text/markdown, text/html, */* — markdown first. If a server responds with markdown, it's used directly. If it gets HTML, it converts client-side via Turndown.

This means if we configure nginx to serve the .html.md files that sphinx-llm already generates when the client requests text/markdown, the agent would get markdown automatically on every fetch — no llms.txt discovery needed, no special hints.

To test this, we'd add something like this to nginx:

# If client prefers markdown and a .md version exists, serve it
if ($http_accept ~* "text/markdown") {
    set $try_md "yes";
}
# ... rewrite to .html.md when available

This is complementary to llms.txt — llms.txt helps with discovery (what pages exist), content negotiation helps with delivery (each page in the right format).

2. Sphinx Show Source

Disabled by canonical_sphinx. It forcibly sets html_copy_source = False and html_show_sourcelink = False — overriding even explicit conf.py settings. So none of our 6 builds have source files in the output.

We could override this with a post-build copy or a later config handler, but it's fighting the theme. The source files would be .rst.txt or .md.txt — raw markup, not rendered.

The value proposition is weaker than content negotiation: the source is in the original markup format (MyST/RST), not clean markdown. And the agent would need to know to look in _sources/ — it's not a well-known convention.

My recommendation

Content negotiation is worth testing as a Condition E — it requires zero agent awareness (Claude Code already does it), uses the .md files we've already generated, and is the direction the industry is heading. It would also show whether the value of llms.txt is in the discovery index or in the markdown format — because content negotiation gives you the format without the index.

The Sphinx source approach is less compelling — it's fighting the canonical_sphinx theme, the output format is less clean, and the agent doesn't know to look for it.
